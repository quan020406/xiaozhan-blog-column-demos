# CSDN 发布检查清单

每篇文章发布前按这份清单检查，保证专栏风格一致。

## 内容检查

- 标题是否是问题导向，而不是课件章节名。
- 开头是否从贯穿项目的具体问题切入。
- 是否解释了这个测试知识点解决什么交付问题。
- 是否包含可复制的表格、清单、模板或步骤。
- 是否有常见误区，帮助新人避坑。
- 是否自然引出下一篇。

## 原创案例检查

- 不直接使用参考课件中的原始案例。
- 不复用课件中的公开网站、接口地址、人物关系和类比。
- 同一篇内只围绕一个主场景展开，避免案例跳来跳去。
- 如果使用默认 CampusHub，模块名、角色名、数据都来自 `case-project/`。

## 工程边界检查

- 不写真实学校、真实用户、真实手机号、真实账号。
- 不写本机绝对路径、内网地址、端口和私有密钥。
- 不伪造 Selenium、JMeter、性能测试执行结果。
- 后续如加入 Demo，先提供可复现命令，再在文章中引用结果。

## 结构检查

- 一级标题只保留文章标题。
- 二级标题承担文章骨架，不堆过长段落。
- 表格字段名清晰，读者能直接复制改造。
- 结尾给出本篇清单和下一篇方向。
