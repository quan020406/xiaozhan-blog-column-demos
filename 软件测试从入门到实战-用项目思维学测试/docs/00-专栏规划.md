# 专栏规划：软件测试从入门到实战

专栏名：**软件测试从入门到实战：用项目思维学测试**

核心思路：不按课件章节机械搬运，而是把测试知识放进同一个虚拟项目中讲。读者每读一篇，都能看到测试活动如何服务于真实项目交付。

## 原创案例原则

专栏统一使用 **CampusHub 校园服务协作平台** 作为贯穿项目。课件中出现过的案例、类比、网站、人物关系和业务场景不直接使用；如遇到相同知识点，改用 CampusHub 的账号、活动、预约、借用、通知、审核等模块重新设计。

## 每篇建议结构

1. 项目问题：从 CampusHub 的具体问题切入。
2. 概念解释：解释本期知识点解决什么问题。
3. 方法拆解：给出步骤、表格或判断标准。
4. 项目示例：用 CampusHub 的功能重新设计案例。
5. 常见误区：说明新人容易踩的坑。
6. 本期清单：给读者一个可以复制使用的模板。

## 首批建议优先写作

优先完成 01-08 期。这一组文章能先把读者带入测试岗位视角，并为后续用例设计、自动化和性能测试打基础。
