# 贯穿项目方案对比

本专栏需要一个能贯穿 26 篇文章的原创项目。它必须同时满足四个条件：

1. 功能足够多，能覆盖需求、用例、Bug、自动化和性能测试。
2. 业务足够直观，零基础读者不需要额外行业背景。
3. 场景不能和参考课件中的案例重复。
4. 后续可以扩展成轻量 Demo 或测试素材，不依赖真实个人数据。

## 方案 A：CampusHub 校园服务协作平台

一句话：把校园里的活动报名、场地预约、设备借用和通知审核集中到一个平台。

| 维度 | 内容 |
| --- | --- |
| 核心用户 | 学生、社团负责人、后勤管理员、系统管理员 |
| 核心模块 | 账号中心、活动中心、场地预约、设备借用、消息通知、后台审核 |
| 适合讲解 | 需求分析、流程测试、权限测试、Bug 管理、Selenium、JMeter |
| 性能场景 | 热门活动开放报名、预约高峰、活动列表查询 |
| 自动化场景 | 登录、活动报名、筛选查询、后台审核 |
| 优点 | 贴近学生读者，功能边界清楚，容易理解 |
| 风险 | 校园业务略偏教学，需要写得项目化，避免像课程作业 |

推荐指数：5/5。适合当前专栏定位，尤其适合面向测试新人和校招读者。

## 方案 B：FitReserve 健身房预约与会员平台

一句话：会员通过平台预约课程、购买私教课、签到入场，门店后台管理教练、课程和会员权益。

| 维度 | 内容 |
| --- | --- |
| 核心用户 | 会员、教练、门店前台、店长 |
| 核心模块 | 登录注册、课程预约、会员卡、签到核销、教练排班、后台统计 |
| 适合讲解 | 边界值、判定表、状态流转、支付前置校验、权限测试 |
| 性能场景 | 热门课程抢约、门店大屏签到、课程列表查询 |
| 自动化场景 | 预约课程、取消预约、会员权益校验 |
| 优点 | 业务生活化，流程清晰，状态转换丰富 |
| 风险 | 涉及支付时要控制范围，避免把专栏带偏到交易系统 |

推荐指数：4/5。比校园项目更商业化，适合想写得更像真实业务系统的路线。

## 方案 C：BookNest 图书借阅与阅读社区

一句话：用户在线检索图书、预约借阅、续借归还，并在社区发布书评和书单。

| 维度 | 内容 |
| --- | --- |
| 核心用户 | 普通读者、图书管理员、内容审核员 |
| 核心模块 | 图书检索、预约借阅、续借归还、逾期提醒、书评审核、库存管理 |
| 适合讲解 | 等价类、边界值、场景法、异常流程、数据一致性 |
| 性能场景 | 图书检索、热门图书预约、批量消息提醒 |
| 自动化场景 | 搜索、预约、续借、书评审核 |
| 优点 | 数据规则直观，适合讲测试用例设计 |
| 风险 | 性能和并发冲突场景不如报名/预约类项目强 |

推荐指数：3.5/5。适合偏“用例设计和业务规则”的专栏风格。

## 方案 D：TaskFlow 团队任务协作平台

一句话：学生团队或小公司用它管理项目任务、成员分工、评审流转和进度报表。

| 维度 | 内容 |
| --- | --- |
| 核心用户 | 项目负责人、成员、评审人、管理员 |
| 核心模块 | 项目空间、任务看板、评论附件、状态流转、成员权限、统计报表 |
| 适合讲解 | 权限矩阵、状态机、回归测试、灰盒思维、接口自动化 |
| 性能场景 | 看板加载、批量任务更新、报表查询 |
| 自动化场景 | 创建任务、流转状态、权限校验、评论上传 |
| 优点 | 很适合讲工程协作和测试开发 |
| 风险 | 对零基础读者来说，任务系统的业务吸引力不如校园/健身场景 |

推荐指数：4/5。适合后续想把专栏延伸到测试平台、质量工程和协作流程。

## 方案 E：ClinicLite 轻量门诊预约平台

一句话：患者预约科室和医生，前台处理挂号，医生查看排队和就诊记录。

| 维度 | 内容 |
| --- | --- |
| 核心用户 | 患者、前台、医生、管理员 |
| 核心模块 | 科室查询、医生排班、预约挂号、取消改约、叫号提醒、后台排班 |
| 适合讲解 | 场景法、判定表、权限测试、时间边界、可靠性测试 |
| 性能场景 | 放号高峰、医生列表查询、提醒消息发送 |
| 自动化场景 | 挂号、取消、改约、排班查看 |
| 优点 | 业务严谨，天然适合讲质量风险 |
| 风险 | 医疗场景容易引入合规和敏感信息，写作时要抽象处理 |

推荐指数：3.5/5。适合严肃业务风格，但对入门专栏略重。

## 推荐选择

首选：**方案 A CampusHub 校园服务协作平台**。

理由：

- 读者门槛最低，适合测试入门专栏。
- 模块足够覆盖 26 篇内容。
- 可以自然承接 Selenium 和 JMeter 实战。
- 不需要使用真实商业数据或敏感场景。
- 和参考课件中的原案例重合度低。

备选：如果你希望专栏更像商业项目，选择 **方案 B FitReserve**；如果你希望后续更容易过渡到测试平台和质量工程，选择 **方案 D TaskFlow**。
