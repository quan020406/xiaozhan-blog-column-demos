# CampusHub Testing Lab UI 设计规范

本文档是 CampusHub Testing Lab 前端开发的 UI 设计基准。后续登录页、学生端、管理端、测试资产、自动化报告和性能报告页面，都应先按本文档实现，再根据真实页面反馈微调。

相关参考文件：

- `docs/ui-design-style-options.html`：风格选择稿。
- `docs/00-贯穿项目设定.md`：业务角色、模块和示例需求。
- `docs/01-CampusHub项目开发实施方案.md`：技术栈、功能范围和开源边界。

## 1. 设计结论

项目采用双模式视觉体系：

| 模式 | 风格定位 | 默认用途 |
| --- | --- | --- |
| 浅色模式 | shadcn 极简黑白 | 默认模式，覆盖学生端、管理端、文档型页面、表单、列表、详情、Bug 管理、测试用例管理 |
| 深色模式 | 深色实验室 | 可切换模式，覆盖同一业务页面；在自动化执行报告、性能测试、日志追踪、CI 结果页中作为重点展示模式 |

默认启动为浅色模式。深色模式不是单独产品线，而是同一组件体系下的主题切换结果。任何新页面都必须同时检查浅色和深色表现。

## 2. 产品气质

CampusHub 不是商业运营后台，也不是纯课程宣传页。它是一个面向测试学习者的可运行教学项目，因此视觉目标是：

- 清晰可信：界面像真实系统，方便读者理解测试对象。
- 克制专业：减少装饰，避免营销式大图、漂浮卡片和过度渐变。
- 证据优先：用状态、表格、日志、时间线和报告承载测试证据。
- 易于测试：组件结构稳定，文本明确，便于 Selenium 和人工测试定位。
- 学习友好：关键状态、错误原因、下一步操作要显性表达。

不采用的方向：

- 不做大面积彩色渐变背景。
- 不做复杂 3D、拟物、毛玻璃作为主界面。
- 不用装饰性图形替代真实业务信息。
- 不把所有内容放进多层卡片中。
- 不让深色模式只改背景色，必须同步调整边框、层级、状态色和图表。

## 3. 主题策略

### 3.1 浅色模式

浅色模式使用 shadcn 极简黑白风格：

- 背景以白色和浅灰为主。
- 字体层级明确，标题强，正文克制。
- 圆角小，边框清楚，阴影少用。
- 主操作使用黑色或近黑色按钮。
- 只在状态、风险、结果和趋势上使用彩色。

适用页面：

- 登录页。
- 学生端首页。
- 活动报名、场地预约、设备借用、BookNest 图书借阅。
- 管理端审核、账号权限、消息通知、审计记录。
- 测试用例、Bug 报告、需求评审清单、接口调试页面。

### 3.2 深色模式

深色模式使用深色实验室风格：

- 背景使用近黑蓝，不使用纯黑。
- 面板使用深灰蓝，与背景保留可见边界。
- 强调色使用琥珀黄，表达运行、监控和实验感。
- 成功、失败、阻塞、告警必须比浅色模式更高对比。
- 日志、运行状态、图表、报告在深色模式中优先保证可读性。

适用页面：

- 自动化执行报告。
- JMeter 性能测试报告。
- CI 执行结果。
- Selenium 截图与失败日志。
- 接口调用追踪。
- 管理端夜间使用场景。

### 4.4 字体

字体栈：

```css
font-family: Inter, "Segoe UI", "PingFang SC", "Microsoft YaHei", Arial, sans-serif;
```

字号规范：

| 层级 | 字号 | 行高 | 字重 | 用途 |
| --- | --- | --- | --- | --- |
| Display | 40px | 48px | 700 | 少量介绍页大标题 |
| H1 | 30px | 38px | 700 | 页面标题 |
| H2 | 24px | 32px | 700 | 分区标题 |
| H3 | 18px | 28px | 600 | 卡片、面板标题 |
| Body | 14px | 22px | 400 | 默认正文、表格内容 |
| Small | 13px | 20px | 400 | 辅助说明 |
| Label | 12px | 18px | 600 | 标签、表头、字段名 |

限制：

- 不使用负字距。
- 不按视口宽度动态缩放字号。
- 中文正文不小于 13px。
- 表格、表单、弹窗中的标题不得使用 hero 级字号。

### 4.5 间距

统一使用 4px 基准：

| Token | 值 | 用途 |
| --- | --- | --- |
| `--space-1` | 4px | 图标和文字间距 |
| `--space-2` | 8px | 表单项内部、标签间距 |
| `--space-3` | 12px | 卡片内紧凑间距 |
| `--space-4` | 16px | 默认容器 padding |
| `--space-5` | 20px | 表单组、工具栏 |
| `--space-6` | 24px | 页面分区 |
| `--space-8` | 32px | 主内容上下分隔 |

### 4.6 圆角和阴影

| Token | 值 | 用途 |
| --- | --- | --- |
| `--radius-xs` | 4px | 标签、进度条 |
| `--radius-sm` | 6px | 输入框、小按钮 |
| `--radius-md` | 8px | 卡片、表格容器、弹窗 |
| `--radius-lg` | 12px | 登录容器、少量展示卡 |

默认卡片圆角不超过 8px。深色模式中的运行报告面板可使用 10px 到 12px，但不得全站泛用大圆角。

阴影只用于弹窗、下拉、抽屉和浮层。普通卡片优先使用边框，不使用大阴影。

## 5. 信息架构

### 5.1 应用外壳

桌面端：

- 左侧导航宽度：248px。
- 折叠导航宽度：64px。
- 顶部栏高度：56px。
- 主内容最大宽度不限，但内容区左右 padding 为 24px。
- 页面标题区和内容区之间保留 20px 到 24px。

移动端：

- 左侧导航改为抽屉。
- 顶部栏保留页面标题、返回、主题切换和用户菜单。
- 表格默认降级为卡片列表或横向滚动表格。

### 5.2 导航分组

建议导航结构：

| 分组 | 页面 |
| --- | --- |
| 总览 | 首页、学习路线、今日任务 |
| 校园服务 | 活动中心、场地预约、设备借用、消息通知 |
| BookNest | 图书检索、借阅记录、预约队列、逾期提醒 |
| 管理后台 | 审核中心、用户权限、配置管理、审计日志 |
| 测试资产 | 需求清单、测试用例、Bug 报告、接口集合 |
| 测试执行 | Selenium 执行、JMeter 报告、CI 结果、截图证据 |

### 5.3 页面标题区

每个业务页必须有：

- 页面标题。
- 一句用途说明。
- 主要操作按钮。
- 面包屑只在三层以上页面使用。

示例：

```text
活动报名
查看可报名活动，验证名额、截止时间和重复报名规则。
[新建活动] [导出报名名单]
```

## 6. 组件规范

### 6.1 按钮

按钮类型：

| 类型 | 用途 | 浅色 | 深色 |
| --- | --- | --- | --- |
| Primary | 页面主动作 | 黑底白字 | 琥珀底深字 |
| Secondary | 次级动作 | 浅灰底黑字 | 深灰底白字 |
| Outline | 工具栏动作 | 白底边框 | 透明底边框 |
| Ghost | 低优先级动作 | 透明 | 透明 |
| Destructive | 删除、驳回、重置 | 红色 | 亮红色 |

规则：

- 一个页面首屏最多一个 Primary。
- 危险操作必须二次确认。
- 按钮文字使用动词加对象，例如 `提交预约`、`驳回申请`。
- 图标按钮必须有 tooltip 或 `aria-label`。

### 6.2 表单

表单用于登录、预约、借阅、审核和 Bug 报告。

规则：

- 标签放在输入框上方，避免左右标签在窄屏拥挤。
- 必填项使用文字或星号明确标注。
- 错误文案放在字段下方，不只改变边框颜色。
- 同一表单中主提交按钮固定在底部右侧或表单尾部。
- 日期、时间、数量字段必须展示输入限制。

字段状态：

| 状态 | 表现 |
| --- | --- |
| 默认 | 边框 `--input` |
| 聚焦 | 边框和 ring 使用 `--ring` |
| 错误 | 边框和说明使用失败色 |
| 禁用 | 背景使用 `--muted`，文本降低对比但仍可读 |
| 只读 | 背景轻灰，保留文本高对比 |

### 6.3 表格

表格是本项目的核心组件，用于用例、Bug、审核、借阅记录和日志。

规则：

- 表头固定使用 12px 到 13px，字重 600。
- 行高建议 44px 到 52px。
- 默认显示 8 到 12 列以内，超出列进入详情抽屉。
- 状态列使用 Badge，不能只用颜色点。
- 操作列固定在右侧，最多展示 3 个可见操作，其余放入更多菜单。
- 空表格必须说明原因和下一步。

表格必备能力：

- 搜索。
- 筛选。
- 排序。
- 分页。
- 批量操作。
- 列可见性设置，后期可选。

### 6.4 卡片

卡片只用于独立信息块：

- 指标卡。
- 学习任务卡。
- 报告摘要。
- 移动端列表项。
- 空状态和引导入口。

限制：

- 不在卡片内再套完整卡片。
- 不把普通页面分区做成漂浮卡片。
- 卡片默认使用 1px 边框和 8px 圆角。

### 6.5 Badge 和状态标签

Badge 文字必须表达状态，不使用单纯色块。

推荐标签：

- `通过`
- `失败`
- `阻塞`
- `待审核`
- `已驳回`
- `已借出`
- `已逾期`
- `库存不足`
- `回归中`

### 6.6 弹窗与抽屉

弹窗用于短流程确认，抽屉用于详情查看和审核。

规则：

- 删除、驳回、重置使用弹窗确认。
- 审核详情、Bug 详情、用例详情使用右侧抽屉。
- 抽屉宽度：桌面端 520px 到 720px。
- 抽屉内底部固定操作区，避免滚动后找不到按钮。

### 6.7 Toast 和 Alert

Toast 用于短反馈，Alert 用于需要用户阅读的状态。

Toast：

- 成功操作显示 2 到 3 秒。
- 失败操作显示明确原因。
- 不用 Toast 承载长说明。

Alert：

- 用于账号锁定、预约冲突、库存不足、性能报告环境说明。
- 必须可读，不能只用图标和颜色。

### 6.8 空状态

空状态结构：

```text
标题：暂无待审核预约
说明：当前没有学生或社团提交新的场地预约。
操作：返回预约列表
```

不使用空泛文案，例如 `暂无数据` 作为唯一说明。

### 6.9 图标

实现时优先使用 `lucide` 系列图标。图标用于增强识别，不替代文本。

常用映射：

| 场景 | 图标语义 |
| --- | --- |
| 搜索 | Search |
| 新建 | Plus |
| 编辑 | Pencil |
| 删除 | Trash |
| 导出 | Download |
| 通过 | Check |
| 失败 | X |
| 告警 | TriangleAlert |
| 日志 | ScrollText |
| 报告 | ChartNoAxesCombined |

## 7. 页面模板

### 7.1 登录页

浅色模式：

- 左侧或顶部展示项目名和一句定位。
- 登录表单居中或右侧放置。
- 输入项：学号/账号、密码、角色提示。
- 错误状态展示账号锁定和剩余时间。

深色模式：

- 背景可使用深色实验室质感。
- 登录卡片仍要保持高对比。
- 不使用低对比玻璃拟态。

### 7.2 首页

首页用于回答三件事：

- 今天我需要处理什么。
- 目前系统质量风险在哪里。
- 下一步学习或测试任务是什么。

模块：

- 任务概览。
- 待审核事项。
- 最近 Bug。
- 脚本通过率。
- 学习路径入口。

### 7.3 学生业务页

适用于活动报名、场地预约、设备借用、图书借阅。

标准结构：

```text
页面标题区
筛选工具栏
列表或卡片
详情抽屉
提交或取消操作
结果反馈
```

### 7.4 管理审核页

标准结构：

```text
待处理指标
筛选条件
审核列表
右侧详情抽屉
通过、驳回、退回补充材料
审计日志
```

审核操作必须展示：

- 申请人。
- 申请对象。
- 时间。
- 规则校验结果。
- 历史记录。
- 操作影响。

### 7.5 测试用例管理页

核心字段：

- 用例编号。
- 模块。
- 前置条件。
- 操作步骤。
- 预期结果。
- 优先级。
- 状态。
- 关联需求。

页面要求：

- 支持按模块、状态、优先级筛选。
- 步骤和预期结果在详情抽屉中展示。
- 列表中只保留可扫描字段。

### 7.6 Bug 报告页

核心字段：

- Bug 编号。
- 标题。
- 严重程度。
- 优先级。
- 复现步骤。
- 实际结果。
- 预期结果。
- 环境。
- 截图或日志证据。
- 关联用例。

页面要求：

- 严重程度和优先级不得混用颜色。
- 失败证据使用附件区、日志块或截图区。
- 状态流转清楚展示：新建、处理中、已修复、回归中、关闭、重新打开。

### 7.7 自动化执行报告页

深色模式优先。

模块：

- 执行批次。
- 总用例数、通过、失败、跳过、耗时。
- 失败用例列表。
- 截图证据。
- 控制台日志。
- 环境信息。

要求：

- 日志区使用等宽字体。
- 失败信息可复制。
- 截图缩略图可打开大图。
- 不伪造运行结论，报告只展示实际数据或示例数据。

### 7.8 性能测试报告页

深色模式优先。

模块：

- 测试目标。
- 环境说明。
- 并发配置。
- 响应时间指标。
- 错误率。
- 吞吐量。
- 关键观察。

图表规则：

- 折线图用于趋势。
- 柱状图用于接口对比。
- 表格用于原始指标。
- 深色模式中网格线降低亮度，文字保持可读。

## 8. 响应式规则

断点：

| 名称 | 宽度 |
| --- | --- |
| Mobile | `< 640px` |
| Tablet | `640px - 1023px` |
| Desktop | `>= 1024px` |
| Wide | `>= 1440px` |

规则：

- Mobile 下导航使用抽屉。
- Mobile 下工具栏允许换行，筛选项进入折叠区域。
- Mobile 下表格优先转卡片，确需表格时允许横向滚动。
- Desktop 下主内容宽度自适应，不强制居中窄列。
- 固定格式组件必须有稳定尺寸，避免 hover、加载和错误状态造成布局跳动。

## 9. 可访问性

最低要求：

- 正文和背景对比度不低于 WCAG AA。
- 所有交互控件可通过键盘访问。
- 焦点态必须可见。
- 图标按钮必须提供 `aria-label`。
- 表单错误必须文本说明。
- Toast 不作为唯一错误反馈。
- 深色模式不能依赖纯色差表达状态。

键盘规则：

- `Tab` 在页面主交互区按视觉顺序移动。
- 弹窗打开后焦点进入弹窗，关闭后回到触发按钮。
- 抽屉和下拉可用 `Esc` 关闭。

## 10. 开发落地规则

项目技术栈建议为 Vue 3 + Vite + TypeScript。即使最终使用 Element Plus，也应使用本文档的 token 覆盖视觉表现。

### 10.1 CSS 结构

建议结构：

```text
frontend/src/styles/
  tokens.css
  theme.css
  base.css
  components.css
  element-overrides.css
```

职责：

| 文件 | 职责 |
| --- | --- |
| `tokens.css` | 变量定义 |
| `theme.css` | light/dark 主题切换 |
| `base.css` | reset、字体、body、链接、滚动条 |
| `components.css` | 项目自定义组件样式 |
| `element-overrides.css` | Element Plus 样式覆写 |

### 10.2 主题切换

根节点使用：

```html
<html data-theme="light">
```

切换后：

```html
<html data-theme="dark">
```

规则：

- 用户选择写入 `localStorage`。
- 未选择时跟随系统偏好，但默认演示截图使用浅色。
- 自动化测试中允许通过 URL 参数或测试前置脚本固定主题。

### 10.4 Selenium 友好约定

关键元素必须提供稳定测试选择器：

```html
<button data-testid="activity-submit-button">提交报名</button>
<tr data-testid="bug-row-BUG-001">...</tr>
<section data-testid="selenium-report-summary">...</section>
```

规则：

- `data-testid` 使用英文小写和连字符。
- 不使用随机 ID。
- 不把测试定位依赖在 CSS hash、图标或动态文案上。
- 重要状态变化要有可断言文本。

## 11. 文案规范

文案要帮助学习者理解系统行为。

按钮：

- 使用 `提交预约`，不用 `确定`。
- 使用 `驳回申请`，不用 `拒绝`。
- 使用 `导出用例`，不用 `下载`。

错误：

- 使用 `活动名额已满，无法继续报名。`
- 不使用 `操作失败` 作为唯一提示。

空状态：

- 使用 `暂无待回归 Bug，修复后的问题会出现在这里。`
- 不使用 `暂无数据` 作为唯一提示。

报告：

- 使用 `本次结果仅代表当前本地环境。`
- 不写无法验证的性能结论。

## 12. 截图与文章配图规则

用于 README、CSDN 文章或专栏配图时：

- 优先使用浅色模式截图，便于读者阅读。
- 自动化、性能、日志、CI 内容可使用深色模式截图。
- 截图中不得出现真实个人信息。
- 截图必须使用虚拟账号和示例数据。
- 截图内容要和文章讲解点一致，不放无关装饰页面。

## 13. 验收清单

每个新页面提交前检查：

- 浅色模式可读。
- 深色模式可读。
- 主要按钮、表单、表格、状态标签符合本规范。
- 关键错误状态有文字说明。
- 空状态有下一步操作。
- 移动端不横向溢出，除明确允许横向滚动的表格。
- 关键交互有 `data-testid`。
- 弹窗、抽屉和下拉支持键盘关闭。
- 没有多层卡片嵌套。
- 没有大面积装饰渐变和无意义背景图。
