6 种 UI 风格候选

点击任意方案可以固定选择。每个方案都按 CampusHub 的教学项目场景评估，而不是只看视觉效果。

A. 可信企业后台

Ant Design、Element Plus、TDesign 取向。强调确定性、可扫描、低学习成本，适合登录、审核、表格、测试用例和 Bug 管理。

推荐首选
CampusHubAdmin
预约待审24
用例覆盖86%
  • 色彩蓝色主色、绿色成功、橙色警告、红色失败，符合测试状态表达。
  • 组件顶部筛选、左侧菜单、数据表格、抽屉详情、步骤条和状态标签。
  • 适合教学项目第一版、开源 README 截图、Selenium 稳定定位。

B. Bento 学习工作台

以模块化卡片组织学习路线、任务状态和项目进展。相比传统后台更亲和，适合给测试新人建立“从任务进入知识点”的路径。

学习感强
Learning LabSprint 01
本周任务12
完成文章02
  • 色彩低饱和绿、墨色正文、暖白背景，降低学习疲劳。
  • 组件学习卡片、进度模块、任务清单、推荐下一步、章节入口。
  • 适合门户首页、课程地图、个人学习看板和文章配套页面。

C. Liquid Glass 空间层级

吸收 Apple HIG 材料层级和近年 Glassmorphism 趋势，用半透明面板制造现代感。适合项目宣传、登录页和轻量仪表盘。

展示出彩
QA StudioGlass
性能任务08
缺陷修复19
  • 色彩紫、蓝、暖橙少量叠加，背景必须保留足够对比。
  • 组件毛玻璃导航、浮层卡片、轻阴影、渐隐背景和重点指标。
  • 适合登录页、项目介绍页、演示视频封面和少量概览页。

D. shadcn 极简黑白

高对比、少装饰、强排版，接近 shadcn/ui 和现代开发者工具风格。适合代码、接口、测试脚本和学习文档密集场景。

开发者感
TestOpsCLI
脚本集16
接口用例42
  • 色彩黑白灰为主，少量状态色只用于结果、风险和操作反馈。
  • 组件命令面板、表格、Tabs、代码块、空状态、轻量弹窗。
  • 适合开源项目、技术读者、接口文档和测试资产管理。

E. Fluent AI 协作台

借鉴 Fluent 2 与 Copilot 类产品的柔和层级，把“任务、建议、证据、下一步”放在同一界面，适合后续扩展 AI 测试助手。

面向扩展
QA CopilotAssist
建议操作05
证据片段31
  • 色彩微软蓝、浅灰、白色面板，强调柔和高效而非强视觉冲击。
  • 组件建议面板、右侧详情、聊天式辅助、任务卡和证据时间线。
  • 适合AI 辅助测试、需求评审、Bug 诊断和脚本生成。

F. 深色实验室

偏测试实验台、监控面板和自动化运行结果。深色背景配高亮状态，适合性能压测、脚本执行、日志追踪等场景。

  • 组件日志流、运行状态、性能曲线、错误列表、用例批次结果。
  • 适合JMeter、Selenium 执行报告、CI 页面和实时监控。

后续设计规范落地项

选定风格后，建议把下面 4 类内容固化成项目级 UI 规范，再进入 Vue 页面开发。

Design Tokens

主色、状态色、文本色、边框、阴影、圆角、间距、字号和层级。

Core Components

按钮、输入框、选择器、表格、标签、弹窗、抽屉、步骤条、空状态。

Testing States

待测、执行中、通过、失败、阻塞、已修复、回归中、已关闭。

Page Patterns

登录、首页、列表筛选、详情审核、用例管理、Bug 报告、执行报告。
