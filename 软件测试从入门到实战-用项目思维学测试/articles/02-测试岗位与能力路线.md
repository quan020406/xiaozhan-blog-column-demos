# 02｜测试工程师、测开、开发到底有什么区别？

## 开篇问题

以默认方案 CampusHub 为例：活动报名模块要上线，团队里有产品、开发、测试、测试开发和运维。新人最容易困惑的问题是：测试工程师是不是只负责点页面？测试开发是不是比测试高级？测试为什么还要懂代码？

这一篇要回答：**不同岗位不是按高低排序，而是按交付职责分工。**

## 本期目标

- 理解测试工程师、测试开发、开发工程师的工作边界。
- 知道测试为什么需要学习代码、数据库、网络和工具。
- 给测试新人一条可执行的能力路线。

## 正文大纲

### 1. 同一个功能，不同角色看问题的角度不同

以“用户完成一次关键业务操作”为例：

| 角色 | 关注重点 |
| --- | --- |
| 产品 | 谁能操作、什么时候截止、异常时如何提示 |
| 开发 | 接口、数据库、状态流转、异常处理 |
| 测试 | 需求是否可验证、流程是否正确、异常是否覆盖 |
| 测试开发 | 自动化脚本、测试数据构造、测试工具和质量平台 |
| 运维 | 发布、监控、容量、故障恢复 |

### 2. 测试工程师的核心工作

- 参与需求评审，提前发现不可测或不清晰的需求。
- 设计测试用例，覆盖正常、异常和边界场景。
- 执行测试，提交清晰可复现的缺陷。
- 做回归测试和发布风险评估。
- 输出测试报告和质量结论。

### 3. 测试开发的核心工作

- 把高价值、重复执行的用例自动化。
- 封装公共测试工具和数据准备能力。
- 建设接口测试、UI 自动化、性能测试或质量平台。
- 让测试活动更稳定、更高效、更可追溯。

### 4. 为什么测试也要懂开发知识

- 看懂接口和日志，定位问题更快。
- 理解数据库状态，判断数据是否真的正确。
- 编写自动化脚本，减少重复劳动。
- 评估技术风险，而不是只描述页面现象。

### 5. 新人能力路线

| 阶段 | 重点能力 | 练习方式 |
| --- | --- | --- |
| 入门 | 需求理解、用例设计、Bug 描述 | 围绕贯穿项目写功能测试用例 |
| 进阶 | 接口、数据库、Linux、日志分析 | 用接口和数据验证页面结果 |
| 自动化 | Selenium、接口自动化、测试框架 | 把稳定流程转成脚本 |
| 专项 | 性能、安全、质量平台 | 选择一个方向做深 |

## 本期清单：测试新人学习顺序

1. 先学会读需求。
2. 再学会写清楚测试点和测试用例。
3. 继续练 Bug 复现、描述和沟通。
4. 补接口、数据库、HTTP、Linux 基础。
5. 选择自动化或性能测试作为进阶方向。
