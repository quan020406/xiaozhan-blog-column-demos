# 学习路线

| 阶段 | 学习主题 | CampusHub 入口 | 产出物 |
| --- | --- | --- | --- |
| 1 | 认识测试对象 | 账号、活动、BookNest、场地、设备、通知 | 功能清单与风险清单 |
| 2 | 需求分析 | 报名名额、借阅上限、预约时段、设备库存 | 需求评审问题列表 |
| 3 | 用例设计 | 等价类、边界值、判定表、场景法 | 手工测试用例表 |
| 4 | 缺陷管理 | 库存不一致、重复报名、权限越权 | Bug 报告样例 |
| 5 | 接口测试 | `/api/auth/login`、`/api/books`、`/api/devices` | Postman 集合 |
| 6 | UI 自动化 | 登录、报名、预约、借用、通知已读 | Selenium 脚本与截图 |
| 7 | 性能入门 | 健康检查、活动、场地、设备、图书查询 | JMeter 脚本与 JTL 摘要 |
| 8 | 质量看板 | QA Command Center | 看板 JSON、截图、执行摘要 |

建议学习顺序是先读 `README.md` 跑通项目，再看 `test-assets/test-cases/`，最后运行 Selenium、JMeter 或性能探针，把结果刷新到 QA Console。
