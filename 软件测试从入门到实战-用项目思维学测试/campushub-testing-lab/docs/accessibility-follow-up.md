# CampusHub Testing Lab 可访问性后续清单

本文记录 CampusHub Testing Lab 1.0 阶段一质量底线修复后，仍建议在后续阶段继续补齐的可访问性问题。阶段一已经完成基础语义修复，包括图标按钮命名、提示语义、分栏键盘操作、输入控件标签和基础 dialog 语义。

## 后续范围

后续阶段重点不是重做页面结构，而是在现有 Vue 组件和视觉风格上补齐键盘、焦点和屏幕阅读器体验。

| 优先级 | 问题 | 建议处理 | 验收标准 |
| --- | --- | --- | --- |
| P1 | 详情抽屉打开后没有主动聚焦 | 打开用例详情或缺陷详情时，把焦点移动到抽屉标题或关闭按钮 | 只用键盘点击“查看详情/查看复盘”后，下一次 Tab 进入抽屉内部 |
| P1 | 详情抽屉关闭后没有恢复焦点 | 记录触发按钮，关闭后把焦点还给原按钮 | 关闭抽屉后，继续按 Tab 不会从页面顶部重新开始 |
| P1 | 抽屉缺少 Esc 关闭 | 给用例详情抽屉、缺陷详情抽屉补充 Escape 键关闭 | 键盘用户按 Esc 能关闭当前抽屉，且不会触发其他页面动作 |
| P1 | 抽屉打开时背景仍可被键盘访问 | 对背景内容做 inert 或实现焦点陷阱 | 抽屉打开后，Tab 和 Shift+Tab 只在抽屉内循环 |
| P2 | 浮动指南是非模态 dialog，但缺少焦点恢复 | 打开指南时记录触发按钮，关闭后恢复焦点 | 从“打开指南”进入并关闭后，焦点回到“打开指南”按钮或等价入口 |
| P2 | 浮动指南拖拽手柄缺少键盘等价操作 | 为浮动指南位置或尺寸提供键盘可操作方案，或明确保留为鼠标增强能力 | 不使用鼠标也能调整指南尺寸，或能通过现有大小下拉完成主要阅读需求 |
| P2 | 移动端底部操作栏和浮动指南可能存在焦点顺序冲突 | 检查移动端 Tab 顺序，必要时调整 DOM 顺序或 `tabindex` | 375px 宽度下，从顶部到主内容再到底部操作的焦点顺序符合视觉阅读顺序 |
| P3 | 复杂表格缺少标题或描述关联 | 给用例矩阵、文章证据矩阵补充 `caption` 或 `aria-describedby` | 屏幕阅读器进入表格时能先读出表格用途 |
| P3 | 状态徽标只依赖文字和颜色混合表达 | 检查 `StatusBadge` 在所有场景下是否有明确文本 | 不看颜色也能判断成功、失败、阻塞、警告等状态 |

## 建议实施顺序

1. 先做 P1：详情抽屉焦点进入、焦点恢复、Esc 关闭和焦点陷阱。这部分直接影响键盘用户是否能完成“查看详情-关闭-继续浏览”的闭环。
2. 再做 P2：浮动指南和移动端焦点顺序。这部分影响实操跟做体验，但可以在不改变业务流程的前提下逐步补齐。
3. 最后做 P3：表格描述和状态表达统一。这部分适合和后续组件整理或无障碍审计一起处理。

## 建议验证方式

后续每轮可访问性修复至少执行：

```powershell
cd campushub-testing-lab\frontend
npm run build
```

并在浏览器中手工验证：

| 场景 | 验证动作 | 预期结果 |
| --- | --- | --- |
| 用例详情抽屉 | Tab 到“查看详情”，按 Enter 打开，再按 Esc 关闭 | 焦点进入抽屉，关闭后回到原“查看详情”按钮 |
| 缺陷详情抽屉 | Tab 到“查看复盘”，按 Enter 打开，再按 Shift+Tab | 焦点不会跑到抽屉外部背景 |
| 浮动指南 | 在实操工作台打开指南，调整大小并关闭 | 控件有名称，关闭后焦点位置可预测 |
| 分栏控件 | Tab 到分栏 separator，按左右方向键 | Testing guide 和 Article 比例按步进变化，且读屏可获得当前值 |
| 移动端 | 以窄屏宽度从页面顶部连续 Tab | 焦点顺序和可见界面顺序一致 |

## 暂不纳入后续清单的事项

- 不引入新的 UI 组件库或可访问性依赖，除非后续明确做组件体系升级。
- 不为了无障碍修复重构 `App.vue`，除非页面交互复杂度已经阻碍维护。
- 不改变业务 API、示例数据、文章数据和既有测试定位属性。
