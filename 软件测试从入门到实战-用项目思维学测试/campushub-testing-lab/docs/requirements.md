# CampusHub Testing Lab 01-26 需求说明

本文档按 `docs/07-CampusHub-Lite-PRD.md` 的当前 01-26 口径维护。目标是让读者先操作普通业务页面，再基于同一项目学习需求分析、用例设计、Bug 描述、接口测试、Selenium 和 JMeter。

## 1. 角色范围

| 角色 | 用途 | 示例账号 |
| --- | --- | --- |
| 学生 | 登录、活动报名、BookNest 借阅、场地预约、设备借用、通知和我的记录 | `student01 / campus123` |
| 学生 | 数据隔离、自动化默认账号和反向场景 | `student02 / campus123` |
| 锁定学生 | 登录失败和账号状态边界 | `student_locked / campus123` |
| 管理员 | 管理员简表、审核任务和权限差异 | `admin01 / campus123` |

## 2. 功能范围

| 编号 | 模块 | 需求 | 验收口径 |
| --- | --- | --- | --- |
| L-001 | 登录 | 用户输入正确用户名和密码后可以登录 | 页面显示登录成功和当前用户信息 |
| L-002 | 登录 | 用户名为空时给出明确提示 | 页面提示“请输入用户名” |
| L-003 | 登录 | 密码为空时给出明确提示 | 页面提示“请输入密码” |
| L-004 | 登录 | 用户名或密码错误应被拒绝 | 后端返回 401，页面给出失败提示 |
| L-005 | 登录 | 锁定账号不能登录 | 后端返回 403，页面给出失败提示 |
| L-006 | 登录 | 学生和管理员登录后展示不同可操作范围 | 学生不能处理审核，管理员可进入审核任务 |
| A-001 | 活动报名 | 学生可以查看活动列表 | 页面展示活动名称、地点、报名人数、容量和状态 |
| A-002 | 活动报名 | 未满员且开放的活动允许报名 | 报名成功后人数加 1，并进入我的记录 |
| A-003 | 活动报名 | 满员、关闭或重复报名应被拒绝 | 前端按钮不可用或后端返回明确失败原因 |
| A-004 | 活动报名 | 重复报名、满员活动、关闭活动应被拒绝 | 前端按钮不可用或后端返回明确失败原因 |
| A-005 | 活动报名 | 学生可以取消自己的有效报名 | 取消后人数和我的记录同步更新 |
| A-006 | 活动报名 | 活动报名操作应生成通知和审计记录 | 通知列表出现业务反馈，后端写入审计日志 |
| B-001 | BookNest | 学生可以查看并检索图书 | 关键词检索后列表刷新 |
| B-002 | BookNest | 有库存图书允许借阅 | 借阅成功后库存减 1，并进入我的记录 |
| B-003 | BookNest | 无库存、重复借阅、逾期续借应被拒绝 | 操作失败时给出明确原因 |
| B-004 | BookNest | 未逾期且未超过续借次数的借阅记录允许续借 | 到期时间延后，续借次数增加 |
| B-005 | BookNest | 逾期或已归还记录不能续借 | 后端返回明确失败原因 |
| B-006 | BookNest | 归还成功后借阅状态更新，图书库存恢复 | 我的记录和图书列表同步 |
| B-007 | BookNest | 学生可以在我的记录查看借阅、续借和归还状态 | 我的记录展示当前账号借阅状态 |
| R-001 | 场地预约 | 学生可以查看场地列表、容量、状态和活跃预约数 | 页面展示场地基本信息和预约状态 |
| R-002 | 场地预约 | 可用场地允许提交未来日期和合法时段的预约 | 合法预约生成待审核记录 |
| R-003 | 场地预约 | 维护中场地、过去日期、非法时段和重叠预约应被拒绝 | 后端返回明确冲突或参数错误 |
| R-004 | 场地预约 | 预约成功后生成待审核记录、通知和审核任务 | 我的记录、通知和管理员审核任务同步 |
| D-001 | 设备借用 | 学生可以查看设备列表、分类、总数、可借数量和状态 | 页面展示设备基本信息和库存状态 |
| D-002 | 设备借用 | 可用设备允许提交合法数量和日期区间的借用申请 | 合法申请生成待审核记录 |
| D-003 | 设备借用 | 无库存、数量非法、日期非法和重复申请应被拒绝 | 页面或接口给出明确失败原因 |
| D-004 | 设备借用 | 借用成功后生成待审核记录、通知和审核任务 | 我的记录、通知和管理员审核任务同步 |
| N-001 | 消息通知 | 学生可以查看自己的通知 | 通知列表只展示当前账号数据 |
| N-002 | 消息通知 | 学生可以将自己的通知标记为已读 | 已读状态更新 |
| N-003 | 消息通知 | 跨用户通知不可被读取或修改 | 跨用户通知返回 404 |
| M-001 | 我的记录 | 学生可以查看自己的报名、借阅、场地和设备记录 | 业务操作后状态同步 |
| AD-001 | 管理员入口 | 管理员可以查看活动报名统计、图书库存和异常样例 | `admin01` 登录后可查看简表 |
| AD-002 | 后台审核 | 管理员可以查看待审核任务 | 管理员入口展示审核任务列表 |
| AD-003 | 后台审核 | 管理员可以通过或驳回待审核任务 | 审核状态、审核人、备注和时间更新 |
| AD-004 | 后台审核 | 学生访问管理员审核动作时应被拒绝 | 学生处理审核返回 403 |

## 4. 非目标和边界

- QA Command Center 不作为首页主入口。
- 不接入真实学校、真实账号、真实手机号、真实邮箱、短信、邮件或支付。
- 不把未运行过的 Selenium、JMeter 或性能探针结果写成真实结论。
- 不把本地 `docs/`、`articles/`、`assets/images/` 作为 GitHub 上传内容。
