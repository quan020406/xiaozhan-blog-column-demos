# CampusHub 第一阶段需求说明

本文档描述第一阶段工程骨架的需求边界。第一阶段目标是让读者能够在本地启动一个真实项目壳子，并看到后续测试文章会围绕哪些模块展开。

## 1. 角色范围

| 角色 | 第一阶段用途 | 后续阶段用途 |
| --- | --- | --- |
| 学生 | 作为示例数据展示 | 登录、报名活动、借阅图书 |
| 社团负责人 | 作为示例数据展示 | 发布活动、查看报名名单 |
| 图书管理员 | 作为示例数据展示 | 管理 BookNest 图书和借阅 |
| 后勤管理员 | 作为示例数据展示 | 审核场地和设备申请 |
| 系统管理员 | 作为示例数据展示 | 管理权限和审计日志 |

## 2. 第一阶段功能需求

| 编号 | 需求 | 验收口径 |
| --- | --- | --- |
| P1-R-001 | 后端应用可以启动 | 执行 `mvn spring-boot:run` 后，`/api/health` 返回 `UP` |
| P1-R-002 | 后端启动时加载 H2 示例数据 | `/api/overview` 返回用户、活动、图书、场地和设备统计 |
| P1-R-003 | 前端应用可以启动 | 执行 `npm run dev` 后浏览器可访问 Vite 页面 |
| P1-R-004 | 前端展示后端概览数据 | 页面展示模块、统计、活动列表和图书列表 |
| P1-R-005 | 项目 README 可指导本地运行 | README 包含环境要求、启动命令、默认地址和示例账号 |
| P1-R-006 | 文档说明需求和数据结构 | `docs/requirements.md` 和 `docs/data-dictionary.md` 可读 |

## 3. 第一阶段非目标

- 不实现真实登录鉴权。
- 不实现活动报名、取消报名、图书借阅、续借和归还接口。
- 不实现后台审核页面。
- 不引入真实短信、邮件、支付或学校组织数据。
- 不提供性能测试结论。

## 4. 后续测试教学映射

| 模块 | 可承接文章内容 |
| --- | --- |
| 账号中心 | 等价类、边界值、权限矩阵、登录自动化 |
| 活动中心 | 场景法、判定表、冒烟测试、性能测试 |
| BookNest | 图书借阅状态流转、边界值、数据一致性 |
| 后台审核 | 权限测试、审计日志、Bug 定位证据 |

## 5. 示例数据边界

- 示例账号、书名、活动名和场地名均为虚拟教学数据。
- 密码字段只在 README 中作为演示约定，数据库当前保留 `password_hint`，不用于真实认证。
- 后续实现登录时应改为哈希密码或测试专用认证策略，不能把明文密码作为生产方案。
