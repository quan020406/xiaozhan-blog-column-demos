# Bug 报告样例：归还图书后可借库存未恢复

| 字段 | 内容 |
| --- | --- |
| Bug 编号 | CH-BUG-001 |
| 标题 | 归还 BookNest 图书后，图书列表可借数量未恢复 |
| 严重程度 | Major |
| 优先级 | P1 |
| 发现版本 | `0.2.0-SNAPSHOT` |
| 模块 | BookNest 图书借阅 |
| 环境 | Windows 11，JDK 17，Node.js 24，Chrome，H2 内存库 |
| 测试账号 | `student01 / campus123` |

## 前置条件

- 后端通过 `mvn spring-boot:run` 启动。
- 前端通过 `npm run dev` 启动。
- `student01` 已登录。
- 图书《接口测试任务手册》当前可借数量大于 0。

## 复现步骤

1. 打开 `http://localhost:5173`。
2. 使用 `student01 / campus123` 登录。
3. 在 BookNest 区域点击《接口测试任务手册》的“借阅”。
4. 记录图书列表中的可借数量。
5. 在“我的借阅”中点击该借阅记录的“归还”。
6. 再次查看图书列表中的可借数量。

## 期望结果

归还成功后，该图书可借数量应增加 1，借阅记录状态变为 `RETURNED`。

## 实际结果

可借数量没有增加，导致库存与借阅记录不一致。

## 附加信息

这是教学用 Bug 报告样例，用于说明字段完整性和复现步骤写法；当前实现已按期望行为处理库存恢复。

## 建议定位方向

- 检查归还接口是否更新 `book.available_copies`。
- 检查前端归还后是否重新加载图书列表。
- 检查接口是否返回成功但数据库更新失败。
