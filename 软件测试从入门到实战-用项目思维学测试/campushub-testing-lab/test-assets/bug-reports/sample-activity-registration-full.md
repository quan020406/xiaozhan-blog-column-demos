# Bug 报告样例：满员活动仍可提交报名请求

| 字段 | 内容 |
| --- | --- |
| Bug ID | CHL-BUG-001 |
| 标题 | 满员活动仍可通过接口提交报名请求，页面和接口口径不一致 |
| 模块 | 活动报名 |
| 严重级别 | Major |
| 优先级 | P0 |
| 发现版本 | CampusHub Testing Lab 0.2.0-SNAPSHOT |
| 测试账号 | `student01 / campus123` |

## 前置条件

1. 后端和前端已启动。
2. 示例数据中存在报名人数等于容量的活动，或通过多次报名把某活动推到满员状态。
3. 当前账号未报名该活动。

## 复现步骤

1. 使用 `student01 / campus123` 登录。
2. 打开“活动列表”，找到已满员活动，记录活动 ID。
3. 如果页面按钮已禁用，使用 Postman 集合中的“活动报名”请求，替换为该活动 ID。
4. 发送报名请求。
5. 再次刷新活动列表和“我的记录”。

## 实际结果

接口仍可能返回报名成功，或报名人数超过活动容量。

## 预期结果

系统应拒绝满员活动报名，返回 `409`，错误原因应明确表达“活动名额已满”；活动人数不能超过容量，“我的记录”不能新增报名记录。

## 影响

该问题会破坏活动容量规则，使边界值测试和后台报名统计失真。它适合作为第一阶段 Bug 报告写作示例，用来训练“前置条件、步骤、实际结果、预期结果和影响”的完整表达。
