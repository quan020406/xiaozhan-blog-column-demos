# 手工测试用例：活动报名与 BookNest 借阅核心闭环

## 测试范围

本用例集覆盖第一版验收要求中的普通学生核心流程：登录、报名活动、借阅图书、续借图书、归还图书。

## 测试环境

| 项 | 说明 |
| --- | --- |
| 后端 | `mvn spring-boot:run` |
| 前端 | `npm run dev` |
| 默认前端地址 | `http://localhost:5173` |
| 示例账号 | `student01 / campus123`、`admin01 / campus123` |
| 数据库 | H2 内存库，启动时加载 `backend/src/main/resources/data.sql` |

## 用例列表

| 用例编号 | 标题 | 前置条件 | 步骤 | 预期结果 | 优先级 |
| --- | --- | --- | --- | --- | --- |
| CH-TC-001 | 学生账号登录成功 | 后端和前端已启动 | 1. 打开首页；2. 输入 `student01` 和 `campus123`；3. 点击“登录” | 页面显示“测试学生一号 登录成功”，活动和图书数据加载 | P0 |
| CH-TC-002 | 学生报名开放活动 | 已使用 `student01` 登录；存在状态为 `OPEN` 的活动 | 1. 在“活动报名”表格选择一条未报名开放活动；2. 点击“报名” | 页面提示“活动报名成功”，该活动状态显示“已报名”，名额数增加 1 | P0 |
| CH-TC-003 | 满员活动不可报名 | 已使用 `student01` 登录；存在状态为 `FULL` 的活动 | 1. 找到状态为 `FULL` 的活动；2. 观察操作按钮 | 报名按钮不可点击，不能产生新报名记录 | P1 |
| CH-TC-004 | 学生借阅有库存图书 | 已使用 `student01` 登录；存在可借数量大于 0 的图书 | 1. 在 BookNest 区域选择一本可借图书；2. 点击“借阅” | 页面提示“图书借阅成功”；“我的借阅”出现新记录；图书可借数量减少 1 | P0 |
| CH-TC-005 | 无库存图书不可借阅 | 已使用 `student01` 登录；存在可借数量为 0 的图书 | 1. 找到可借数量为 0 的图书；2. 观察借阅按钮 | 借阅按钮不可点击，不能产生借阅记录 | P1 |
| CH-TC-006 | 借阅中图书可续借一次 | 已存在状态为 `BORROWED` 且续借次数为 0 的记录 | 1. 在“我的借阅”点击“续借” | 页面提示“图书续借成功”；续借次数变为 `1/1`；应还日期延后 | P0 |
| CH-TC-007 | 已续借图书不可再次续借 | 已存在续借次数为 1 的借阅记录 | 1. 观察“续借”按钮 | 续借按钮不可点击，不能再次续借 | P1 |
| CH-TC-008 | 学生归还借阅中图书 | 已存在状态为 `BORROWED` 的借阅记录 | 1. 在“我的借阅”点击“归还” | 页面提示“图书归还成功”；借阅状态变为 `RETURNED`；图书可借数量增加 1 | P0 |
| CH-TC-009 | 管理员审核任务通过 | 已使用 `admin01` 登录；存在状态为 `PENDING` 的审核任务 | 1. 在“后台审核”选择一条待审核任务；2. 点击“通过” | 页面提示“审核任务已处理”；该任务状态变为 `APPROVED` | P0 |

## 回归关注点

- 报名、借阅、归还都会修改库存或名额，回归时必须同时检查列表和个人记录。
- 登录失败、账号锁定、重复报名、无库存借阅属于后续文章的缺陷和边界案例入口。
