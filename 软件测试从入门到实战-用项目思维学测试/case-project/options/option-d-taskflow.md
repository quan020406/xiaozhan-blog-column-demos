# 方案 D：TaskFlow 团队任务协作平台

## 一句话设定

学生团队或小公司用它管理项目任务、成员分工、评审流转和进度报表。

## 模块

- 项目空间：创建项目、成员邀请、角色管理。
- 任务看板：创建任务、指派负责人、状态流转。
- 评论附件：评论、上传附件、@成员提醒。
- 评审流程：提交评审、通过、驳回、重新提交。
- 权限管理：管理员、负责人、成员、访客权限。
- 报表统计：任务完成率、延期任务、成员工作量。

## 适配文章

- 需求分析：一句“任务能流转”拆成清晰状态和权限。
- 判定表：角色、任务状态、操作者是否本人共同决定按钮权限。
- 回归测试：状态流转改动后影响多个入口。
- 自动化：创建任务、流转状态、权限校验。
- 测试开发：后续适合扩展接口自动化和质量平台。

## 选择理由

工程味更强，适合后续进阶到测试开发、接口自动化和质量工程。但作为入门专栏，业务亲和力略弱。
