# 方案 B：FitReserve 健身房预约与会员平台

## 一句话设定

会员通过平台预约课程、购买私教课、签到入场，门店后台管理教练、课程和会员权益。

## 模块

- 账号会员：注册、登录、会员等级、权益有效期。
- 课程预约：团课预约、取消预约、候补排队。
- 教练排班：教练课程表、请假、替课。
- 签到核销：扫码签到、迟到规则、旷课记录。
- 订单权益：课包购买、次数扣减、退款申请。
- 门店后台：课程配置、会员管理、数据统计。

## 适配文章

- 用例设计：会员等级、课程容量、预约时间边界。
- 判定表：会员类型、课程状态、剩余次数共同决定是否可预约。
- Bug 管理：取消预约未返还次数、签到状态错误。
- 自动化：登录、预约、取消、权益校验。
- 性能测试：热门课程开放预约瞬间的并发请求。

## 选择理由

商业项目感更强，读者容易联想到真实 SaaS 或小程序。但如果引入支付，需要明确只讲支付前校验，避免偏离软件测试入门主线。
