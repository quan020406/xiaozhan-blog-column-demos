# 方案 A：CampusHub 校园服务协作平台

## 一句话设定

把校园里的活动报名、场地预约、设备借用和通知审核集中到一个平台。

## 模块

- 账号中心：登录、退出、找回密码、角色权限。
- 活动中心：活动发布、报名、取消报名、名单导出。
- 场地预约：查询空闲时段、提交预约、取消预约、超时释放。
- 设备借用：库存查询、借用申请、归还确认、损坏登记。
- 消息通知：站内信、短信提醒、邮件提醒。
- 后台审核：审核通过、驳回、批量处理、操作日志。

## 适配文章

- 基础认知：活动报名上线前如何判断质量是否可信。
- 用例设计：登录、报名、预约、借用等输入和流程都容易拆解。
- Bug 管理：取消预约后名额未释放、重复报名、通知未发送。
- 自动化：登录、活动列表、报名流程、后台审核。
- 性能测试：热门活动开放报名、活动列表查询高峰。

## 选择理由

这是当前最推荐方案。读者门槛低，功能范围完整，既能讲手工测试，也能自然过渡到 Selenium 和 JMeter。
