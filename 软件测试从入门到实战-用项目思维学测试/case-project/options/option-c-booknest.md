# 方案 C：BookNest 图书借阅与阅读社区

## 一句话设定

用户在线检索图书、预约借阅、续借归还，并在社区发布书评和书单。

## 模块

- 账号中心：读者登录、管理员登录、权限区分。
- 图书检索：关键词、分类、作者、馆藏状态。
- 借阅预约：预约、借出、续借、归还、逾期提醒。
- 库存管理：馆藏数量、损坏登记、下架维护。
- 阅读社区：书评、点赞、举报、审核。
- 消息提醒：到期提醒、预约到书提醒、审核结果通知。

## 适配文章

- 等价类：关键词、书名、作者、ISBN 等输入规则。
- 边界值：借阅天数、续借次数、书评字数。
- 场景法：预约成功、无库存、逾期未还、归还异常。
- Bug 管理：库存数量不一致、逾期提醒重复发送。
- 自动化：检索、预约、续借、书评审核。

## 选择理由

规则清晰，适合重点讲测试用例设计和数据一致性。并发和性能场景可以讲，但冲击力弱于报名、预约类项目。
