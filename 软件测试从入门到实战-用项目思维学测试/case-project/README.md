# 贯穿项目素材区

这里存放《软件测试从入门到实战：用项目思维学测试》的贯穿项目设定。当前阶段先提供多个候选方案，待你确认后再把需求、测试用例、Bug 报告、自动化脚本和性能素材全部统一到选定方案。

## 候选方案

| 方案 | 项目 | 适合方向 | 文件 |
| --- | --- | --- | --- |
| A | CampusHub 校园服务协作平台 | 入门友好、校招读者、全流程覆盖 | [option-a-campushub.md](./options/option-a-campushub.md) |
| B | FitReserve 健身房预约与会员平台 | 商业项目感、预约和权益规则 | [option-b-fitreserve.md](./options/option-b-fitreserve.md) |
| C | BookNest 图书借阅与阅读社区 | 用例设计、数据规则和流程异常 | [option-c-booknest.md](./options/option-c-booknest.md) |
| D | TaskFlow 团队任务协作平台 | 权限、状态机、测试开发进阶 | [option-d-taskflow.md](./options/option-d-taskflow.md) |
| E | ClinicLite 轻量门诊预约平台 | 严肃业务、预约和可靠性风险 | [option-e-cliniclite.md](./options/option-e-cliniclite.md) |

详细对比见：[../docs/00-贯穿项目方案.md](../docs/00-贯穿项目方案.md)。

## 后续可扩展内容

- `requirements/`：按模块沉淀需求说明。
- `test-cases/`：存放手工测试用例模板。
- `bug-reports/`：存放缺陷报告示例。
- `automation/`：后续如需要，可加入 Selenium 或接口自动化示例。
- `performance/`：后续如需要，可加入 JMeter 脚本说明。

当前阶段只初始化项目设定，不放真实个人信息、真实学校数据、真实接口地址或本机环境配置。
