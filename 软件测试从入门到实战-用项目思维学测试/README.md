# 软件测试从入门到实战：用项目思维学测试

这是一个面向测试新人、转岗学习者和校招实习准备者的测试专栏目录。专栏参考软件测试课程知识体系，但不复用课件中的原始案例、类比和叙述方式，统一使用原创贯穿项目来讲解。

当前初始化阶段提供多个贯穿项目候选，默认推荐 **CampusHub 校园服务协作平台**。你确认最终方案后，再把文章正文、用例模板和后续 Demo 统一锁定到对应项目。

## 专栏定位

- 目标读者：零基础测试学习者、准备测试岗位面试的学生、希望补齐测试体系的开发者。
- 写作方式：每篇围绕一个项目问题展开，用测试思维拆解，再给出清单、模板或可复用方法。
- 内容范围：测试基础、需求分析、缺陷管理、测试用例设计、测试分类、Web 自动化、性能测试和 JMeter 入门。
- 案例边界：所有案例围绕 CampusHub 重新设计，不照搬课件中的生活类比、网站示例、邮箱注册、搜索引擎、博客系统或汽车性能类比。


## 贯穿项目

当前推荐使用 CampusHub，它包含账号登录、活动报名、场地预约、设备借用、消息通知和后台审核等模块，足够覆盖测试基础、用例设计、Bug 管理、自动化和性能测试。贯穿项目候选和专栏规划文档保留在本地 `docs/`，不上传 GitHub。

## 写作约定

- 每篇文章都从 CampusHub 的一个具体功能或质量问题切入。
- 概念解释不堆定义，优先回答“为什么需要这个测试动作”。
- 每篇至少保留一个可复制的表格、清单或模板。
- 不写无法验证的夸张结论，不伪造工具执行结果、性能数据或截图。
- 后续如补充 Demo，代码、数据和文章草稿应分目录存放，避免混在根目录。
