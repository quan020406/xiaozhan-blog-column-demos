# CSDN 发布检查清单

## 内容

- 标题包含明确技术词：Spring Boot、MyBatis-Plus、Redisson、AST、CI/CD、SOLID。
- 开头 200 字说明读者能解决什么问题。
- 每篇只解决一个主问题，代码与结论可独立验证。
- 不使用“百分之百”“史上最强”等无法证明的标题。
- 性能数字来自真实测试，并注明机器、并发、请求数和数据库条件。

## 排版

- 二级标题组织主流程，三级标题解释步骤。
- 代码块标明语言；命令区分 PowerShell、Bash、Python。
- 图片下方提供图注和可检索的替代文字。
- 长表格只保留关键对比，避免在移动端无法阅读。

## 发布前验证

- 执行 `mvn.cmd test`。
- 执行 `python -m unittest discover -s ai_firm/tests`。
- 执行两种模式压测，并替换文章中的“示例输出”。
- 上传本篇对应封面图和架构图。
- 检查文末上一篇、下一篇链接。
